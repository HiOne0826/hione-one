## 📋 待完成（按顺序执行）

### 步骤 1：在服务器上启动 Webhook 服务

```bash
ssh myserver
cd /root/hione-one

# 启动 Webhook 服务（后台运行）
pkill -f 'python3 webhook.py' 2>/dev/null  # 杀掉旧进程（如果有）
nohup python3 webhook.py > webhook.log 2>&1 &

# 验证是否启动成功
ps aux | grep webhook.py
# 应该能看到 python3 webhook.py 进程

# 查看日志
tail -f webhook.log
```

---

### 步骤 2：配置腾讯云防火墙（重要！）

需要开放 3000 端口让 GitHub Webhook 能访问：

1. 访问腾讯云控制台：https://console.cloud.tencent.com/cvm/instance
2. 找到你的服务器，点击 "安全组" → "配置规则"
3. 添加入站规则：
   - 类型：自定义
   - 来源：`0.0.0.0/0`（或只允许 GitHub IP，可选）
   - 协议端口：`TCP:3000`
   - 策略：允许

---

### 步骤 3：在 GitHub 上配置 Webhook

1. 访问：https://github.com/HiOne0826/hione-one/settings/hooks
2. 点击 "Add webhook"
3. 填写：
   - **Payload URL**: `http://124.156.136.199:3000/webhook`
   - **Content type**: `application/json`
   - **Secret**: `5f613a1fe12b3a8b5bb478b776c17db3`
   - **Which events**: 选择 `Just the push event`
4. 点击 "Add webhook"

## ❓ 故障排查

### Webhook 服务没启动？
```bash
ssh myserver
ps aux | grep webhook.py
# 如果没运行，重新启动：
cd /root/hione-one
nohup python3 webhook.py > webhook.log 2>&1 &
```

### GitHub Webhook 显示 "We couldn't deliver this payload"？
检查：
1. 腾讯云防火墙是否开放 3000 端口
2. Payload URL 是否正确（http://，不是 https://）
3. Secret 是否一致
