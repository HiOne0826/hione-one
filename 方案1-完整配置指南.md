# 方案1：GitHub Webhook 实时自动部署（完整配置）

## 工作原理

```
你本地改代码 → 构建 → 提交推送到 GitHub 
    ↓
GitHub 触发 Webhook
    ↓
服务器 Webhook 接收器收到信号
    ↓
服务器自动拉取 dist 目录并部署
```

**实时！没有 5 分钟延迟！**

---

## 步骤 1：本地配置（已完成）

✅ 修改 `.gitignore`，允许提交 `dist/` 目录

## 步骤 2：服务器上配置 Webhook 接收器

### 2.1 上传 Webhook 接收器

```bash
cd /Users/ericgao/Desktop/hione-one
scp webhook.py myserver:/root/hione-one/
ssh myserver
chmod +x /root/hione-one/webhook.py
```

### 2.2 设置 Webhook 密钥（重要！）

编辑 `/root/hione-one/webhook.py`，修改这一行：
```python
SECRET = b"your-webhook-secret-here"  # 改成你自己的密钥
```

### 2.3 启动 Webhook 接收器

```bash
ssh myserver
cd /root/hione-one

# 用 nohup 后台运行
nohup python3 webhook.py > webhook.log 2>&1 &

# 检查是否运行
ps aux | grep webhook.py
```

### 2.4 设置开机自启动（可选但推荐）

```bash
ssh myserver
crontab -e
```

添加这一行：
```
@reboot cd /root/hione-one && nohup python3 webhook.py > webhook.log 2>&1 &
```

---

## 步骤 3：GitHub 上配置 Webhook

1. 访问：https://github.com/HiOne0826/hione-one/settings/hooks
2. 点击 "Add webhook"
3. 填写：
   - **Payload URL**: `http://124.156.136.199:3000/webhook`
   - **Content type**: `application/json`
   - **Secret**: 你在 `webhook.py` 里设置的密钥
   - **Which events**: `Just the push event`
4. 点击 "Add webhook"

## 故障排查

### Webhook 接收器没启动？
```bash
ssh myserver
ps aux | grep webhook.py
# 如果没运行，重新启动：
cd /root/hione-one
nohup python3 webhook.py > webhook.log 2>&1 &
```

### GitHub Webhook 显示失败？
检查：
1. 服务器安全组/防火墙是否开放 3000 端口
2. Payload URL 是否正确
3. Secret 是否一致
