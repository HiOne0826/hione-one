# hione.one 自动部署指南

## 方案：GitHub Webhook + 服务器脚本

### 步骤 1：服务器上设置

1. **在服务器上克隆项目**
```bash
ssh myserver
cd /root
git clone https://github.com/HiOne0826/hione-one.git
cd hione-one
npm install
```

2. **上传部署脚本**
```bash
# 在本地执行：
scp deploy.sh myserver:/root/hione-one/
ssh myserver
chmod +x /root/hione-one/deploy.sh
```

3. **测试部署脚本**
```bash
ssh myserver
cd /root/hione-one
./deploy.sh
```

### 步骤 2：设置 Webhook 接收器（可选，简单版用 Cron）

#### 方案 A：Cron 定时任务（最简单）

在服务器上设置每 5 分钟检查一次 GitHub：

```bash
ssh myserver
crontab -e
```

添加以下内容：
```
*/5 * * * * cd /root/hione-one && git fetch origin && git rev-parse HEAD > current.txt && git rev-parse origin/main > remote.txt && diff current.txt remote.txt || ./deploy.sh
```

#### 方案 B：Webhook 接收器（推荐，实时）

需要设置一个简单的 Webhook 服务器，参考 `webhook-server.js`

### 步骤 3：GitHub 上配置 Webhook

1. 访问 https://github.com/HiOne0826/hione-one/settings/hooks
2. 点击 "Add webhook"
3. 填写：
   - Payload URL: `http://your-server-ip:3000/webhook`
   - Content type: `application/json`
   - Secret: 你自己的密钥
   - Which events: "Just the push event"
4. 点击 "Add webhook"

## 快速开始（推荐先用 Cron）

如果你不想搞太复杂，直接用 **Cron 定时任务** 就行，每 5 分钟自动检查一次 GitHub，有更新就自动部署。

**简单版：**

```bash
# 在服务器上运行：
ssh myserver
cd /root
git clone https://github.com/HiOne0826/hione-one.git
cd hione-one
npm install
chmod +x deploy.sh
# 添加 Cron 任务
(crontab -l 2>/dev/null; echo "*/5 * * * * cd /root/hione-one && git fetch origin && git rev-parse HEAD > /tmp/current.txt && git rev-parse origin/main > /tmp/remote.txt && diff /tmp/current.txt /tmp/remote.txt || ./deploy.sh") | crontab -
```
